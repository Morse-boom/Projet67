Introduction

C'est quoi Luau ?

🧠 Luau vs Roblox Studio — quelle différence ?

Roblox Studio, c'est le logiciel. C'est l'outil que tu ouvres sur ton PC pour construire ton jeu. Tu y places des pièces, tu modifies des couleurs, tu organises ton monde. C'est comme Minecraft en mode création : tu construis avec des blocs.

Luau, c'est le langage de programmation utilisé dans Roblox Studio. C'est la langue que tu parles pour donner des ordres à ton jeu. Sans Luau, ton jeu est statique — il ne se passe rien.

💡
Analogie : Roblox Studio c'est la scène de théâtre, et Luau c'est le script des acteurs. Sans script, les acteurs restent immobiles. Avec le script, ils savent quoi faire et comment réagir.

Luau est basé sur un autre langage appelé Lua. Roblox a modifié Lua pour le rendre plus rapide et plus sécurisé — d'où le nom Lua + U = Luau.

📄 C'est quoi un script ?

Un script, c'est un fichier texte qui contient des instructions écrites en Luau. Roblox lit ces instructions et les exécute dans le jeu.

Luau
-- Quand un joueur touche la brique, il gagne 10 points
print("Bonjour ! Le script fonctionne.")

Les 3 types de scripts dans Roblox :

📜
Script
S'exécute côté serveur. Il contrôle ce qui se passe pour tout le monde. Par exemple : gérer les points, spawner des ennemis.
🖥️
LocalScript
S'exécute côté client (l'ordi du joueur). Il gère l'interface, les boutons, les effets visuels propres à un joueur.
🧩
ModuleScript
Un script "bibliothèque". Il contient des fonctions que d'autres scripts peuvent utiliser. Il ne s'exécute pas tout seul.

📍 Où mettre les scripts ?

ScriptOù le mettrePourquoi
ScriptServerScriptServiceIl s'exécute automatiquement côté serveur
LocalScriptStarterPlayerScripts ou StarterGuiIl s'exécute pour chaque joueur individuellement
ModuleScriptReplicatedStorage ou ServerScriptServiceAccessible par les autres scripts
⚠️
Erreur classique : mettre un LocalScript dans ServerScriptService. Ça ne marchera pas — Roblox l'ignorera silencieusement. Vérifie toujours où tu mets tes scripts.

📌 Résumé

  • Luau = le langage pour donner des ordres à Roblox
  • Roblox Studio = le logiciel pour construire le jeu
  • Script = fichier texte avec des instructions Luau
  • 3 types : Script (serveur), LocalScript (client), ModuleScript (bibliothèque)

Variables & Types de données

📦 C'est quoi une variable ?

Une variable, c'est une boîte dans laquelle tu ranges une valeur. Tu donnes un nom à la boîte, et tu peux retrouver la valeur plus tard en utilisant ce nom.

📦
Analogie : Imagine une boîte en carton avec une étiquette dessus. L'étiquette, c'est le nom de la variable. Ce qu'il y a dedans, c'est la valeur. Tu peux changer ce qu'il y a dans la boîte à tout moment.
Luau — Variable simple
local nom = "Alex"
print(nom)  -- affiche : Alex
local→ on crée une variable locale
nom→ c'est le nom de la boîte
=→ on met quelque chose dans la boîte
"Alex"→ c'est la valeur : le texte "Alex"
print(nom)→ on affiche le contenu dans la console
Luau — Changer la valeur
local points = 0
print(points)  -- 0
 
points = 10
print(points)  -- 10
 
points = points + 5
print(points)  -- 15

🔒 C'est quoi local ?

local dit : "cette variable existe seulement dans cet endroit du script". Sans local, la variable devient globale — elle peut causer des bugs difficiles à trouver.

✅ Bonne pratique
local x = 5
-- x existe seulement ici
❌ À éviter
x = 5
-- x est globale : risque de bugs
💚
Règle d'or : Toujours écrire local devant tes variables. Sans exception.

🎨 Les 4 types de données

📝
string — texte
Du texte entre guillemets.
local nom = "Roblox"
🔢
number — nombre
Un nombre entier ou décimal.
local vie = 100
✅
boolean — vrai/faux
true ou false, pas de guillemets.
local ok = true
🕳️
nil — rien
nil veut dire "vide". Valeur par défaut.
local x  -- vaut nil

Les Conditions

🤔 C'est quoi une condition ?

Une condition, c'est une question que tu poses à Roblox. Si la réponse est "oui" (true), le code s'exécute. Si c'est "non" (false), il ne s'exécute pas.

🚦
Analogie : C'est comme un feu rouge. Si le feu est vert, tu avances. Sinon, tu attends.
Luau — if / else / elseif
local score = 75
 
if score >= 90 then
    print("Excellent ! 🏆")
elseif score >= 60 then
    print("Bien joué ! 👍")
else
    print("Il faut s'entraîner !")
end
-- Affiche : Bien joué ! 👍

⚖️ Opérateurs de comparaison

SymboleSignificationExemple
==Égal à5 == 5 → true
~=Différent de5 ~= 3 → true
>Supérieur à10 > 5 → true
<Inférieur à3 < 7 → true
>=Supérieur ou égal5 >= 5 → true
<=Inférieur ou égal4 <= 5 → true
⚠️
Erreur très fréquente : confondre = et ==. = assigne une valeur. == compare. Si tu écris if vie = 0 then, Luau va afficher une erreur.
Luau — and / or / not
local vie = 50
local armure = true
 
-- and : les DEUX conditions doivent être vraies
if vie > 0 and armure == true then
    print("Joueur protégé !")
end
 
-- not : inverse la condition
if not armure then
    print("Pas d'armure !")
end
Exercice Système de médaille : or si ≥90, argent si ≥60, bronze si ≥30 
local score = 70
 
if score >= 90 then
    print("🥇 Médaille d'or !")
elseif score >= 60 then
    print("🥈 Médaille d'argent")
elseif score >= 30 then
    print("🥉 Médaille de bronze")
else
    print("Pas de médaille cette fois")
end

📌 Résumé

  • if condition then ... end
  • else → si c'est faux
  • elseif → tester une autre condition
  • == pour comparer, = pour assigner
  • and, or, not pour combiner

Les Boucles

🔁 Boucle for

Utilisée quand tu sais combien de fois répéter.

Luau — for
for i = 1, 5 do
    print("Tour numéro " .. i)
end
-- Affiche : Tour numéro 1, 2, 3, 4, 5
 
-- Compter à rebours
for i = 5, 1, -1 do
    print(i)  -- 5, 4, 3, 2, 1
end

🔄 Boucle while

Répète tant qu'une condition est vraie.

Luau — while
local vie = 100
 
while vie > 0 do
    print("Vie : " .. vie)
    vie = vie - 25
end
print("Le joueur est mort")
 
-- ✅ Boucle infinie SÉCURISÉE avec task.wait()
while true do
    print("Toutes les secondes")
    task.wait(1)
end
🚨
Danger : la boucle infinie ! Une boucle while true do sans task.wait() fait planter Roblox Studio.
BoucleQuand l'utiliser
for i = 1, nTu sais combien de fois répéter
while conditionTant qu'une condition est vraie
repeat untilAu moins une fois, puis vérifier
Exercice Afficher les nombres de 1 à 10 
for i = 1, 10 do
    print(i)
end

📌 Résumé

  • Boucle = répéter du code
  • for i = 1, n do → n fois précises
  • while condition do → tant que c'est vrai
  • Toujours task.wait() dans les boucles infinies !

Les Fonctions

🔧 C'est quoi une fonction ?

Une fonction, c'est un bloc de code que tu définis une fois et que tu peux appeler autant de fois que tu veux.

🍕
Analogie : Imagine une recette de pizza. Tu l'écris une fois. Ensuite, chaque fois que tu veux une pizza, tu suis la recette.
Luau — Fonction avec paramètres et return
-- Définir
local function saluer(prenom)
    print("Bonjour, " .. prenom .. " !")
end
 
-- Appeler
saluer("Alex")   -- Bonjour, Alex !
saluer("Léa")    -- Bonjour, Léa !
 
-- return : renvoyer une valeur
local function additionner(a, b)
    return a + b
end
 
local resultat = additionner(10, 5)
print(resultat)  -- 15
Exercice 1 Fonction multiplier(a, b) qui retourne a × b 
local function multiplier(a, b)
    return a * b
end
print(multiplier(4, 5))  -- 20
Exercice 2 Fonction estMort(vie) qui retourne true si vie ≤ 0 
local function estMort(vie)
    return vie <= 0
end
print(estMort(0))   -- true
print(estMort(50))  -- false

📌 Résumé

  • Fonction = bloc de code réutilisable
  • local function nom() pour la créer
  • nom() pour l'appeler
  • Paramètres = infos qu'on donne à la fonction
  • return = renvoyer une valeur

Les Tables

📋 Table comme liste & comme dictionnaire

🗂️
Analogie : Un classeur avec des tiroirs. Chaque tiroir a un numéro ou un nom et contient quelque chose.
Luau — Liste & Dictionnaire
-- LISTE (index numérique, commence à 1 !)
local fruits = {"pomme", "banane", "cerise"}
print(fruits[1])  -- pomme
print(fruits[2])  -- banane
print("Taille : " .. #fruits)  -- 3
 
-- DICTIONNAIRE (clé/valeur)
local joueur = {
    nom = "Alex",
    vie = 100,
    niveau = 5
}
print(joueur.nom)   -- Alex
print(joueur.vie)   -- 100
joueur.vie = 75    -- modifier
 
-- Parcourir avec ipairs / pairs
for i, fruit in ipairs(fruits) do
    print(i .. ". " .. fruit)
end
 
-- Ajouter un élément
table.insert(fruits, "kiwi")
print(#fruits)  -- 4
⚠️
Attention ! En Luau, les index commencent à 1, pas à 0 comme dans d'autres langages.

📌 Résumé

  • Table = collection de valeurs
  • Liste : {val1, val2} — accès par table[1]
  • Dictionnaire : {cle = val} — accès par table.cle
  • ipairs pour listes, pairs pour dictionnaires
  • #table = nombre d'éléments

Objets & Instances

🧱 C'est quoi une Instance ?

Dans Roblox, tout est une instance. Une brique, un joueur, une lumière, un script — ce sont tous des instances. Chaque instance a un parent et peut avoir des enfants.

Luau — Créer et trouver des instances
-- Créer une nouvelle brique
local brique = Instance.new("Part")
brique.Name = "MaNouvelleBrique"
brique.Parent = game.Workspace  -- toujours en dernier !
 
-- Accéder à un objet existant
local autrebrique = game.Workspace.MaBrique
 
-- Méthode sûre : FindFirstChild
local brique2 = game.Workspace:FindFirstChild("MaBrique")
if brique2 then
    print("Trouvé : " .. brique2.Name)
else
    print("La brique n'existe pas !")
end
⚠️
Important : Assigne toujours .Parent en dernier, après avoir configuré toutes les propriétés. C'est une bonne pratique pour les performances.

📌 Résumé

  • Instance = tout objet Roblox
  • Instance.new("Type") pour créer
  • .Parent pour placer dans la hiérarchie
  • :FindFirstChild("nom") pour chercher sans risque

Les Événements

⚡ Réagir à ce qui se passe dans le jeu

🔔
Analogie : C'est comme une sonnette. Quelqu'un appuie → la sonnette se déclenche → tu ouvres la porte. Dans Roblox : un joueur touche la brique → Touched se déclenche → ton code s'exécute.
Luau — Événements
local brique = game.Workspace.MaBrique
 
-- Quand quelqu'un touche la brique
brique.Touched:Connect(function(partQuiTouche)
    print("Touché par : " .. partQuiTouche.Name)
end)
 
-- PlayerAdded : quand un joueur rejoint
local Players = game:GetService("Players")
 
Players.PlayerAdded:Connect(function(joueur)
    print(joueur.Name .. " a rejoint le jeu !")
end)
ÉvénementSe déclenche quand…
TouchedQuelque chose touche la Part
TouchEndedQuelque chose arrête de toucher
MouseClickLe joueur clique (ClickDetector)
PlayerAddedUn joueur rejoint le serveur
PlayerRemovingUn joueur quitte le serveur
CharacterAddedLe personnage d'un joueur spawn

📌 Résumé

  • Événement = quelque chose qui se passe dans le jeu
  • objet.Evenement:Connect(function() ... end)
  • La fonction s'exécute à chaque déclenchement

Architecture

Client & Serveur

🌐 La distinction fondamentale

🖥️ Serveur (Script)
  • Gère les données officielles
  • Visible par tous les joueurs
  • Place : ServerScriptService
📱 Client (LocalScript)
  • Gère l'interface et les inputs
  • Visible seulement par CE joueur
  • Place : StarterPlayerScripts
🔐
Sécurité : Ne fais jamais confiance au client ! Un joueur malveillant peut modifier son LocalScript. Toujours valider les actions importantes côté serveur.

📌 Résumé

  • Serveur = l'ordinateur de Roblox, voit tout le monde
  • Client = l'ordinateur du joueur, voit seulement lui
  • Script → ServerScriptService
  • LocalScript → StarterPlayerScripts

WaitForChild

⏳ Attendre que les objets soient chargés

Quand Roblox démarre, les scripts et les objets se chargent en même temps. Un LocalScript peut s'exécuter avant que certains objets aient fini de se charger. Si tu accèdes à un objet qui n'est pas encore là, tu obtiens une erreur.

Luau — WaitForChild
-- ❌ RISQUÉ : MaBrique n'est peut-être pas encore chargée
local brique = game.Workspace.MaBrique  -- peut causer une erreur !
 
-- ✅ SÉCURISÉ : attend que MaBrique soit disponible
local brique = game.Workspace:WaitForChild("MaBrique")
 
-- Exemple concret dans un LocalScript
local RS = game:GetService("ReplicatedStorage")
local monEvent = RS:WaitForChild("MonRemoteEvent")
print("RemoteEvent trouvé !")

📌 Résumé

  • Les objets ne sont pas toujours disponibles immédiatement
  • :WaitForChild("nom") attend que l'objet existe
  • Utilise-le dans les LocalScripts surtout

RemoteEvents — Communication

📡 Faire communiquer Client et Serveur

📮
Analogie : C'est comme envoyer une lettre. Le client écrit un message et l'envoie via le RemoteEvent (la poste). Le serveur reçoit la lettre et agit en conséquence.
Luau — Client → Serveur & Serveur → Client
-- === LOCALSCRIPT (Client) ===
local RS = game:GetService("ReplicatedStorage")
local achatEvent = RS:WaitForChild("AchatEvent")
 
-- Envoyer au serveur
achatEvent:FireServer("Epée", 50)
 
-- Recevoir du serveur
notifEvent.OnClientEvent:Connect(function(message)
    print("Message : " .. message)
end)
 
-- === SCRIPT (Serveur) ===
local achatEvent = RS:WaitForChild("AchatEvent")
 
-- Recevoir du client
achatEvent.OnServerEvent:Connect(function(joueur, article, prix)
    print(joueur.Name .. " veut acheter " .. article)
end)
 
-- Envoyer à un joueur précis
notifEvent:FireClient(joueur, "Tu as gagné !")
 
-- Envoyer à TOUS les joueurs
notifEvent:FireAllClients("Le boss spawn !")
DirectionEnvoyerRecevoir
Client → Serveur:FireServer().OnServerEvent:Connect()
Serveur → 1 Client:FireClient(joueur).OnClientEvent:Connect()
Serveur → Tous:FireAllClients().OnClientEvent:Connect()

📌 Résumé

  • RemoteEvent = messager entre client et serveur
  • Toujours dans ReplicatedStorage
  • FireServer / OnServerEvent (côté serveur)
  • FireClient / OnClientEvent (côté client)

Pratique

Comment lire du code

🔍 Méthode en 5 étapes

  1. Commence par le haut — le code s'exécute de haut en bas
  2. Identifie les variables — qu'est-ce qu'on stocke ?
  3. Identifie les fonctions — quelles actions peut-on faire ?
  4. Identifie les événements — qu'est-ce qui déclenche quoi ?
  5. Lis ligne par ligne en te demandant "que fait cette ligne ?"
Luau — Script commenté ligne par ligne
-- LIGNE 1 : On récupère le service Players
local Players = game:GetService("Players")
 
-- LIGNE 2 : On récupère ReplicatedStorage
local RS = game:GetService("ReplicatedStorage")
 
-- LIGNE 3 : On attend que l'événement existe
local pointsEvent = RS:WaitForChild("PointsEvent")
 
-- LIGNE 4 : On définit une fonction qui donne des points
local function donnerPoints(joueur, montant)
    local ls = joueur:FindFirstChild("leaderstats")
    if ls then
        local pts = ls:FindFirstChild("Points")
        if pts then
            pts.Value = pts.Value + montant
        end
    end
end
 
-- LIGNE 5 : Quand le client envoie l'événement…
pointsEvent.OnServerEvent:Connect(function(joueur, montant)
    if montant > 0 and montant <= 100 then
        donnerPoints(joueur, montant)
    end
end)

Pratique

Écrire des Scripts

🚀 Projet : Brique qui téléporte — 4 étapes progressives

Étape 1 — Détecter le toucher
local brique = game.Workspace.BriqueTeleport
brique.Touched:Connect(function()
    print("Touché !")
end)
Étape 2 — Identifier le joueur
local brique = game.Workspace.BriqueTeleport
local Players = game:GetService("Players")
 
brique.Touched:Connect(function(partTouchee)
    local perso = partTouchee.Parent
    local joueur = Players:GetPlayerFromCharacter(perso)
    if joueur then
        print(joueur.Name .. " a touché !")
    end
end)
Étape 3 — Téléporter
local brique = game.Workspace.BriqueTeleport
local Players = game:GetService("Players")
local destination = Vector3.new(0, 10, 0)
 
brique.Touched:Connect(function(partTouchee)
    local perso = partTouchee.Parent
    local joueur = Players:GetPlayerFromCharacter(perso)
    if joueur then
        local root = perso:FindFirstChild("HumanoidRootPart")
        if root then
            root.CFrame = CFrame.new(destination)
        end
    end
end)
Étape 4 — Version finale avec cooldown anti-spam
local brique = game.Workspace.BriqueTeleport
local Players = game:GetService("Players")
local destination = Vector3.new(0, 10, 0)
local cooldowns = {}
 
brique.Touched:Connect(function(partTouchee)
    local perso = partTouchee.Parent
    local joueur = Players:GetPlayerFromCharacter(perso)
    
    if joueur and not cooldowns[joueur] then
        cooldowns[joueur] = true
        
        local root = perso:FindFirstChild("HumanoidRootPart")
        if root then
            root.CFrame = CFrame.new(destination)
        end
        
        task.wait(2)
        cooldowns[joueur] = nil
    end
end)

Pièges

Erreurs Fréquentes

❌
Oublier end — Chaque if, for, while, function doit se fermer avec end.
❌
Confondre = et == — = assigne. == compare. Dans un if, utilise toujours ==.
❌
Oublier local — Sans local, la variable est globale et peut causer des conflits avec d'autres scripts.
❌
Script au mauvais endroit — Un Script serveur dans StarterGui ne fonctionnera pas. Un LocalScript dans ServerScriptService sera ignoré.
❌
Boucle infinie sans task.wait() — while true do sans wait fait planter Roblox Studio.
❌
Faute de frappe dans le nom d'un objet — workspace.mabrique ≠ workspace.MaBrique. Luau est sensible à la casse !
❌
Oublier WaitForChild dans les LocalScripts — L'objet peut ne pas encore exister. Utilise :WaitForChild().
💡
Conseil : Les messages d'erreur sont tes amis — ils te disent exactement où est le problème. Lis-les attentivement.
